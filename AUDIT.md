# 🔍 Audit — portfolio-app-nextjs

> **Stack :** Next.js 16 · React 19 · TypeScript 5 · next-intl · Framer Motion · Vanilla CSS
> **Date :** Avril 2026

---

## 🔐 Sécurité

### Problèmes critiques

| Priorité | Problème | Fichier |
|---|---|---|
| 🔴 CRITIQUE | `proxy.ts` non reconnu par Next.js — le middleware i18n ne s'exécute jamais | `proxy.ts` |
| 🟠 Medium | Clé API exposée côté client via `NEXT_PUBLIC_WEATHER_API_KEY` | `Weather.tsx:16` |
| 🟡 Low | `locale as any` contourne le typage TypeScript | `i18n/request.ts:7` |
| 🟡 Low | Aucun security header configuré (`X-Frame-Options`, `CSP`…) | `next.config.ts` |

### Détails

**1. `proxy.ts` → doit être `middleware.ts`**

Next.js ne reconnaît que le fichier nommé exactement `middleware.ts` à la racine du projet.
En l'état, toute la logique de redirection i18n est silencieusement ignorée en production :
les routes `/`, `/en`, `/fr` ne sont pas interceptées.

```bash
# Fix : renommer le fichier
proxy.ts → middleware.ts
```

**2. Clé API Weather exposée dans le bundle navigateur**

```ts
// ❌ Visible dans les DevTools de n'importe quel visiteur
const API_KEY = process.env.NEXT_PUBLIC_WEATHER_API_KEY
```

Recommandation : créer une route API serveur `/api/weather` avec une clé `WEATHER_API_KEY`
(sans préfixe `NEXT_PUBLIC_`). Le composant fait alors un fetch vers `/api/weather?city=Lille`.

**3. Security headers manquants**

```ts
// À ajouter dans next.config.ts
const nextConfig: NextConfig = {
  async headers() {
    return [{
      source: '/(.*)',
      headers: [
        { key: 'X-Frame-Options', value: 'DENY' },
        { key: 'X-Content-Type-Options', value: 'nosniff' },
        { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
      ],
    }]
  },
}
```

## 🏗️ Architecture

### Points forts

- Séparation claire `app/components/` (globaux) vs `[locale]/features/` (spécifiques à la route)
- Design system en CSS custom properties bien structuré (`tokens.css`)
- `Button` avec `React.memo` + `forwardRef` — production-ready
- Gestion du thème encapsulée proprement dans `ThemeProvider`
- Icon registry pattern extensible et type-safe
- `AbortController` sur le fetch Weather — bonne pratique

### Problèmes

**1. CSS global couplé à la structure de routing**

```css
/* app/styles/index.css */
@import "@/[locale]/minimal/minimal.css" layer(layout); /* ❌ */
```

Un fichier CSS global ne devrait jamais importer depuis une route.
Si la page `minimal` est renommée ou déplacée, le CSS global se casse.

> Fix : déplacer `minimal.css` dans `app/styles/` ou importer le CSS dans le layout de la route.

**2. Page home inutilement en `"use client"`**

```tsx
// app/[locale]/page.tsx
"use client";
import { useParams } from "next/navigation";
const locale = params.locale as string || 'en';
```

La locale est disponible via les `params` async en Server Component.
Rendre cette page côté serveur améliore les performances et le SEO.

**3. Dossier `sections/` vide**

`app/[locale]/minimal/sections/` est vide — vestige non nettoyé.

**4. Alias `@/components/*` redondant dans `tsconfig.json`**

```json
"paths": {
  "@/*": ["./app/*"],             // ✅ couvre déjà tout
  "@/components/*": ["./app/components/*"]  // ❌ redondant
}
```

## 🔎 SEO

### Problèmes

| Priorité | Problème | Fichier |
|---|---|---|
| 🔴 CRITIQUE | Titre et description = `"Create Next App"` | `layout.tsx`, `[locale]/layout.tsx` |
| 🟠 Fort | Pas d'Open Graph ni Twitter Card | `[locale]/layout.tsx` |
| 🟠 Fort | Pas de `sitemap.ts` ni `robots.ts` | — |
| 🟡 Moyen | Pas de balises `hreflang` pour EN/FR | `[locale]/layout.tsx` |
| 🟡 Moyen | Page home rendue côté client → `<h1>` absent du HTML initial | `[locale]/page.tsx` |

### Détails

**1. Métadonnées placeholder**

```tsx
// ❌ Dans les deux layouts
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

// ✅ À remplacer par
export const metadata: Metadata = {
  title: "Ronan Chenu — Creative Developer",
  description: "Portfolio de Ronan Chenu, développeur créatif basé à Lille.",
  openGraph: {
    title: "Ronan Chenu — Creative Developer",
    type: "website",
  },
  twitter: { card: "summary_large_image" },
};
```

**2. Sitemap et robots manquants**

```ts
// app/sitemap.ts
import type { MetadataRoute } from 'next'

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    { url: 'https://ronan-chenu.dev/en', lastModified: new Date() },
    { url: 'https://ronan-chenu.dev/fr', lastModified: new Date() },
    { url: 'https://ronan-chenu.dev/en/minimal', lastModified: new Date() },
    { url: 'https://ronan-chenu.dev/fr/minimal', lastModified: new Date() },
  ]
}
```

```ts
// app/robots.ts
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: '*', allow: '/' },
    sitemap: 'https://ronan-chenu.dev/sitemap.xml',
  }
}
```

**3. `hreflang` absent malgré le site bilingue**

next-intl ne génère pas automatiquement les balises `hreflang`.
Les moteurs de recherche ne savent pas que `/en/minimal` et `/fr/minimal` sont la même page.

## ⚡ Optimisation du Code

### Points forts

- `React.memo` sur `Button` ✅
- `useMemo` sur le lookup icon ✅
- CSS Layers (`@layer reset, tokens, components, layout, utilities`) ✅
- Polices via `next/font` (évite le FOUT) ✅
- Tokens CSS — pas de valeurs magiques ✅

### Bugs & problèmes

**1. Token `font-size-2xl` inexistant** 🐛

```css
/* minimal.css:62 */
font-size: var(--font-size-2xl); /* ❌ n'existe pas */

/* tokens.css s'arrête à --font-size-xxl */
/* Fix : remplacer par */
font-size: var(--font-size-xxl);
```

**2. `line-height` dupliqué dans `reset.css`** 🐛

```css
/* reset.css */
line-height: var(--line-height-normal); /* ← défini */
font-family: inherit;
line-height: inherit; /* ← écrase immédiatement avec inherit ! */
```

**3. Boutons sociaux sans action dans `Header.tsx`** 🐛

```tsx
// ❌ <button> sans onClick ni href — inutilisables, mauvaise sémantique
<Button variant="primary">
  <Icon name="linkedin" />
</Button>

// ✅ Devrait être un lien
<a href="https://linkedin.com/in/..." target="_blank" rel="noopener noreferrer">
  <Icon name="linkedin" />
</a>
```

**4. Double classe `.muted` + `.accent` sur le même élément** 🐛

```tsx
/* Hero.tsx:9 */
<p className="minimal-hero__subtitle muted accent">Creative Developer</p>
/* ❌ muted → color: var(--muted) */
/* ❌ accent → color: var(--accent) */
/* Les deux s'appliquent → la dernière déclarée dans le CSS gagne, comportement indéfini */
```

**5. CSS mort — `.app-home__link` jamais utilisé**

```css
/* home.css:39-48 — style défini mais le JSX utilise .btn directement */
.app-home__link { ... } /* ❌ dead code */
```

**6. `design.css` vide importé globalement**

```css
/* styles/index.css:6 */
@import "design.css" layer(layout); /* → fichier vide = requête HTTP inutile */
```

**7. Framer Motion non lazy-loadé**

Framer Motion pèse ~100KB gzippé. Utilisé uniquement pour des animations `opacity`/`y`
qui pourraient être faites en CSS natif.

```tsx
// Si Framer Motion est conservé, lazy-loader au minimum :
const MotionDiv = dynamic(() =>
  import('framer-motion').then(mod => mod.motion.div), { ssr: false }
)
```
